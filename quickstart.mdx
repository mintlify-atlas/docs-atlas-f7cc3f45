## Deploy your proxy

<Steps>
  <Step title="Fork the repository">
    Go to [github.com/gaowanlu/google](https://github.com/gaowanlu/google) and click **Fork** in the top-right corner. Accept the default settings and confirm the fork. This creates a copy of the repository under your own GitHub account that Vercel will deploy from.
  </Step>

  <Step title="Set your target URL">
    In your forked repository, open `api/proxy.js` and replace the value of `target` with the URL of the website you want to proxy.

    ```js
    const { createProxyMiddleware } = require("http-proxy-middleware");

    module.exports = (req, res) => {
      let target = "https://www.google.com/"; // your website url

      //   if (
      //     req.url.startsWith("/api") ||
      //     req.url.startsWith("/auth") ||
      //     req.url.startsWith("/banner") ||
      //     req.url.startsWith("/CollegeTask")
      //   ) {
      //     target = "http://106.15.2.32:6969";
      //   }

      createProxyMiddleware({
        target,
        changeOrigin: true,
        pathRewrite: {
          // rewrite request path `/backend`
          //  /backend/user/login => http://google.com/user/login
          //   "^/backend/": "/",
        },
      })(req, res);
    };
    ```

    The commented-out `if` block shows how to route different URL prefixes to different upstream targets. Uncomment and adapt it if you need conditional routing.

    Commit the change directly to your fork's default branch before moving on.
  </Step>

  <Step title="Create a Vercel account and connect GitHub">
    Go to [vercel.com](https://vercel.com) and sign up for a free account. During onboarding, or later in **Settings → Integrations**, connect your GitHub account so Vercel can read your repositories.
  </Step>

  <Step title="Import your forked repository">
    From the Vercel dashboard:

    1. Click **Overview** in the top navigation.
    2. Click **Add New... → Project**.
    3. Click **Continue with GitHub** when prompted.
    4. Find your forked repository (e.g., `your-username/google`) and click **Import**.
    5. Leave the build settings at their defaults — no changes are needed.
    6. Click **Deploy**.
  </Step>

  <Step title="Wait for the deployment to complete">
    Vercel will clone your repository, install dependencies, and deploy the serverless function. This typically takes 30–60 seconds. You will see a success screen with a preview URL when it finishes.
  </Step>

  <Step title="Access your proxy">
    Click the URL shown on the deployment success screen (it will look like `your-project.vercel.app`). Your proxy is now live — requests to that domain are forwarded to the target URL you configured in step 2.

    <Tip>
      For a more reliable and branded experience, register a custom domain and add it to your Vercel project under **Settings → Domains**. Configure a CNAME record with your domain registrar pointing to `cname.vercel-dns.com`, then add the corresponding TXT verification record provided by Vercel. Once DNS propagates, your proxy will be accessible at your own domain.
    </Tip>
  </Step>
</Steps>

## Change the Vercel region

The region your serverless function runs in determines the egress IP address Vercel uses when fetching upstream content. To change it, go to your project in the Vercel dashboard, open **Settings → Functions**, and select a different region from the dropdown. Redeploy the project for the change to take effect.

## Next steps

- Review the `pathRewrite` option in `api/proxy.js` if you need to strip path prefixes before requests reach the upstream server.
- Explore [http-proxy-middleware options](https://github.com/chimurai/http-proxy-middleware#options) for additional configuration such as custom headers, SSL verification, and timeout settings.
