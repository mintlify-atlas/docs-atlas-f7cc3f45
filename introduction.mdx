## What is Vercel Website Proxy?

Vercel Website Proxy is a minimal Next.js template that lets you deploy a reverse proxy for any website using Vercel's serverless infrastructure. Once deployed, all incoming requests are forwarded to a target URL of your choice, and responses are passed back transparently to the visitor.

The project is built around a single serverless function (`api/proxy.js`) powered by [`http-proxy-middleware`](https://github.com/chimurai/http-proxy-middleware), with Vercel's rewrite rules routing every request through it.

## How it works

Every request to your Vercel deployment is captured by a catch-all rewrite rule in `vercel.json`:

```json
{
  "rewrites": [
    {
      "source": "/(.*)",
      "destination": "/api/proxy"
    }
  ]
}
```

The rewrite forwards the request to `api/proxy.js`, which uses `http-proxy-middleware` to proxy it to the configured target URL with `changeOrigin` enabled. Optional `pathRewrite` rules let you strip or transform path prefixes before the request reaches the upstream server.

```
Visitor → Vercel edge → api/proxy.js → Target website → api/proxy.js → Visitor
```

## Prerequisites

Before you deploy, make sure you have:

- A **GitHub account** — you will fork the repository to your own account
- A **Vercel account** — free tier is sufficient; sign up at [vercel.com](https://vercel.com)
- A **target URL** — the website you want to proxy
- A **custom domain** (optional, but recommended) — Vercel provides a default `.vercel.app` domain, though you may want your own for reliability and branding
