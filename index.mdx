## How it works

Every request to your Vercel deployment is routed through a single serverless function (`api/proxy.js`) that forwards traffic to your configured target URL using `http-proxy-middleware`. The `vercel.json` rewrite rule ensures all paths are intercepted:

```json vercel.json
{
  "rewrites": [
    {
      "source": "/(.*)",
      "destination": "/api/proxy"
    }
  ]
}
```

<Steps>
  <Step title="Fork the repository">
    Fork [gaowanlu/google](https://github.com/gaowanlu/google) to your GitHub account.
  </Step>
  <Step title="Set your target URL">
    Edit `api/proxy.js` and replace the `target` value with the website you want to proxy.
  </Step>
  <Step title="Deploy to Vercel">
    Import your forked repository in the Vercel dashboard and deploy with one click.
  </Step>
  <Step title="Connect a custom domain">
    Register a domain and configure CNAME records to point to your Vercel deployment.
  </Step>
</Steps>
